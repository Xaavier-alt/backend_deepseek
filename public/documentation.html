API DOCUMENTATION

Enterprise-grade API specifications with full authentication workflows, rate limiting policies, and security requirements. All endpoints undergo penetration testing quarterly by our cybersecurity team.

CORE PLATFORM APIs

GET https://api.xylexgaminginc.onrender.com/v4/games/{game_id}/leaderboards
game_id (string, required) Unique identifier for the game title
season (integer, optional) Competition season ID. Defaults to current season.
limit (integer, optional) Number of entries to return (max 100)
Authentication: Bearer token (OAuth 2.0)
Rate Limit: 100 requests/minute per client ID
Security: TLS 1.3 required, IP whitelisting available for enterprise
POST https://api.xylexgaminginc.onrender.com/v4/identity/verify-session
session_token (string, required) Active player session token
device_fingerprint (string, required) SHA-256 hash of device characteristics
geolocation (object, optional) Player's current coordinates for fraud detection
Authentication: API key + HMAC signature
Rate Limit: 50 requests/minute per API key
Security: Mandatory request signing, replay attack protection
GET https://api.xylexgaminginc.onrender.com/v4/telemetry/metrics
start_time (timestamp, required) ISO 8601 start time (UTC)
end_time (timestamp, required) ISO 8601 end time (UTC)
metrics (array, required) Array of metric identifiers (fps, latency, memory_usage)
segment (string, optional) Player segment filter (new_players, whales, etc.)
Authentication: Enterprise API key only
Rate Limit: 20 requests/hour (data-intensive endpoint)
Security: GDPR-compliant anonymization, PII filtering enforced
